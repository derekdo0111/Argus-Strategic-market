## v0.7.9 — 链接跳转覆盖范围扩展（窄匹配 → 宽匹配）

### 背景
v0.7.8 的 `preprocessCitations` Step 2 正则 `[A-Z][-\w]*\d+` 只匹配引用格式（`[A1]`、`[W-q-3]`），但 `rehype-sanitize` 给**所有**内部锚点 id 加 `user-content-` 前缀。LLM 若生成 `[abc](#abc)`、`[some-link](#some-link)`、`[REF_001](#ref_001)`、`[中文](#中文)` 等非引用格式链接，href 不加前缀 → 与 id 不匹配 → 点击无跳转。

### 修复
- `ReportViewer.tsx` Step 2 正则从窄匹配 `/\[([A-Z][-\w]*\d+)\]\(#(?!user-content-)([-\w]+)\)/g` → 宽匹配 `/\[([^\]]+)\]\(#(?!user-content-)([^)]+)\)/g`
- 现在**任意** `[文字](#锚点)` 格式的 markdown 内部链接都会被加前缀

### 测试
- `citation-jump.test.tsx` 新增 8 个边界测试：纯小写、连字符、下划线、中文、数字锚点、外部链接不受影响

## v0.7.11 — 股池 QRV 评分栏不显示分数 (根本修复)

### 背景
用户反馈：做完个股分析后股池 QRV 栏目还是不显示分数。上轮 v0.7.7 已经加了 scores 回填逻辑（`stocks.py` L282-297），但数据源 `qrv_analysis.json` 中根本没有 `scores` 字段。

### 根因
`qrv_agent.py` 写完 `qrv_analysis.json` 时，JSON 只包含 `meta`/`gate_status`/`llm_raw_response`/`tokens_used`/`truncated`，**没有 `scores`**。LLM 确实输出了「综合打分卡」表格（如 Q1=8, V3=9, 综合=7.7），但这些数字嵌在 markdown 文本里未被解析。

### 修复
- `qrv_agent.py` 新增 `_parse_scores()` 静态方法 (~50行)：用正则从 LLM markdown 输出的综合打分卡表格提取 10 维度分数 + 综合总分 + 自动算 Q/R/V 加权均分
- `analyze_async()` 写 JSON 前调 `_parse_scores()`，scores 随 JSON 写入
- 已有的 `stocks.py` 回填逻辑现在终于能拿到 scores 数据，前端即时显示

### 向后兼容
- 已分析个股需**重新分析**才能在 JSON 中生成 scores（不重新分析 → scores=None → `"—"`，无破坏）
- `_parse_scores()` 对 N/A 行自动跳过，无表格时返回 None
