## v0.7.6 — 置信度质量加权 + 前端锚点/TOC 修复

### 背景
用户反馈四个问题：1) 置信度全是 HIGH 无区分度 2) 引用 `[W-q-1](#w-q-1)` 裸文本显示无法点击 3) TOC 子标题点击跳错父标题 4) 估值方法不合理（暂不改）

### 方案
按 rules.md 7 步流程执行。

#### 1. 置信度质量加权 (coordinator.py)
- 新增 3 个确定性打分函数（零 LLM）：
  - `_source_credibility(url)` — 基于域名分类（官方1.0/权威0.8/研报0.7/主流媒体0.5/聚合0.3/自媒体0.1）
  - `_info_density(content)` — 数字+单位密度（≥5→1.0, ≥2→0.7, ≥1→0.4）
  - `_recency(text)` — 年份时效性（1年内1.0, 1-3年0.5, 3年+0.2）
- 每条 snippet 打总分 0~3，按质量总分判定模块置信度
- `_tavily_search` snippet 构建 + 模块级置信度改用质量总分
- 老缓存兼容：`quality_score` 新字段可选，下游兜底

#### 2. 前端三修 (ReportViewer.tsx)
- **2a**: `preprocessCitations` 正则加 `(?!\()` 负向前瞻 → Markdown 链接 `[W-q-1](#w-q-1)` 不被炸毁
- **2b**: `rehype-sanitize` schema 允许 `a[id]`/`a[name]` → 锚点跳转生效
- **4**: TOC 子项 `onClick`：`item.id`→`sub.id` + `scrollToSection` 加 h3 DOM 查找
